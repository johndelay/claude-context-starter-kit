# Audit Preparation Guide

What to gather and organize before auditor meetings.

---

## 30 Days Before Audit

### Evidence Collection
- [ ] Pull current user access lists from all critical systems
- [ ] Export vulnerability scan results (last 12 months)
- [ ] Gather remediation evidence for closed vulnerabilities (tickets with verification comments)
- [ ] Collect access review records (quarterly reviews)
- [ ] Screenshot current MFA/SSO configuration
- [ ] Export change management evidence (PR merge logs, deployment records)
- [ ] Locate most recent incident response test/tabletop exercise
- [ ] Verify backup restoration test is documented

### Gap Check
- [ ] Review SOC2_CHECKLIST.md — any "No" or "Partial" entries need a remediation plan or accepted risk documentation
- [ ] Check that all policies have been reviewed within the past 12 months
- [ ] Verify vendor risk assessments are current

---

## 1 Week Before Audit

- [ ] Organize evidence by control category (CC6, CC7, CC8, CC9)
- [ ] Brief relevant team members (who might the auditor interview?)
- [ ] Prepare a "state of security" summary for the auditor kickoff
- [ ] List any known gaps with remediation plans and timelines

---

## During Audit

- [ ] Designate one person as auditor liaison (single point of contact)
- [ ] Log all auditor requests and responses (for tracking and next year's prep)
- [ ] Respond to requests within 24-48 hours
- [ ] If you don't have evidence for something, say so honestly — don't fabricate

---

## After Audit

- [ ] Review findings and draft remediation plan
- [ ] Update SOC2_CHECKLIST.md with any new requirements
- [ ] Schedule remediation work into upcoming sprints
- [ ] Add lessons learned to your CLAUDE.md
